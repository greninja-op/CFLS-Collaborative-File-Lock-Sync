Product focus

Make the next edit an informed decision.

CFLS is intentionally small: it does not replace your editor, your Git host, or your team. It gives all three a shared coordination signal.

Install and try it

Choose the route that matches your role.

Start with the local playground for a visual demo, or follow the role-based team setup for a real shared repository.

Fastest visual demo

Run a Host plus Alice, Bob, and Carol on one machine.

Use this before filming your demo. It launches the local coordination scenario and lets you open separate editor windows to show the team flow.

pnpm install
pnpm playground

What you will see: a local Host and three simulated teammate environments. Open the generated workspaces in separate VS Code windows for the clearest recording.

Admin path

Create a team identity, then start the coordination Host.

Run these commands inside the Git repository that your team shares. The Host can begin on a laptop for a demo and move to an always-on server later.

.\cfls.exe admin-init --team my-team
.\cfls.exe host --url wss://0.0.0.0:8730
.\cfls.exe invite alice <alice-device-public-key>

Important: 0.0.0.0 is only the listening address. Teammates join using this machine's reachable LAN address or DNS name. Use a real TLS certificate for a non-demo deployment.

Teammate path

Register this laptop, receive an invitation, and start the local Agent.

Each teammate uses their own device key. This MVP expects the same Git remote, branch, and base revision as the Host's coordination session.

.\cfls.exe join --host wss://HOST-IP:8730 --name alice --team my-team
.\cfls.exe id
.\cfls.exe connect <invitation>
.\cfls.exe agent --insecure-tls

Demo-only flag: --insecure-tls is for a self-signed local Host. Do not use it for a production Host with a valid certificate.

Supported editor extension

Install the packaged extension, then open the repository while the local Agent is running.

The extension auto-discovers the Agent through a local, per-session configuration file. VS Code and Kiro are the supported editor paths today.

code --install-extension release/cfls-coordination.vsix
kiro --install-extension release/cfls-coordination.vsix

Use --force only when you intentionally want to replace an installed extension version. The extension's current MVP UI is the CFLS status bar and coordination-status command.

Source build

Build the monorepo and package the Windows CLI yourself.

This is the route for contributors or teams that want to make their own internal build. It requires Node.js and pnpm.

pnpm install
pnpm -r build
pnpm -C apps/cli package:win
pnpm -C apps/vscode-extension package:vsix

The built executable is intended for a GitHub Release or internal distribution; it is not automatically included in this static website.

Clear by design

Coordination without inventing another source of truth.

CFLS should make parallel work calmer, not add another place to wonder where code lives.

Does CFLS replace Git?+

No. Git remains responsible for source code, commits, branches, reviews, and remote hosting. CFLS adds real-time coordination context around that workflow.

Does CFLS upload my source code?+

The architecture is designed to analyze dependencies locally and share coordination metadata. Your source files continue to move through Git, not the coordination Host.

Can the Host run on one teammate's laptop?+

Yes, for a local demo or a small co-located session. For an always-on team setup, move the same Host configuration to a stable machine or VPS with proper TLS and backups.

What is ready now and what is next?+

The MVP demonstrates Host/Agent coordination, editor activity, soft lock signals, and device invitations. Dependency ingestion, MCP transport, and stronger protected-path enforcement are the next product hardening steps.
