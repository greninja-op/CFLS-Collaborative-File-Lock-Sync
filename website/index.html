How it works

A tiny agent on each machine talks to one coordination host. Your editor (and any MCP-connected AI agent) reads the shared picture. Files still travel through git.

1

Host

One person (or a small VPS) runs the coordination host — the single source of truth about who's touching what.

2

Agent

Each teammate runs a lightweight agent that watches their folder and exposes a loopback API.

3

Editor + MCP

The VS Code / Kiro extension shows live presence, locks and risk. AI agents read the same data over MCP.

4

Git

CFLS shares metadata only. Actual file bytes move through git — optionally auto-synced per user.

Install

Grab the editor extension and the CLI. Pick your editor below — copy the command, paste it into your terminal.

1 · Download cfls-coordination.vsix. 2 · Open a terminal and run:

code --install-extension cfls-coordination.vsix --force

Then open your repo — the CFLS status bar appears automatically once the agent is running.

1 · Download cfls-coordination.vsix. 2 · Run:

kiro --install-extension cfls-coordination.vsix --force

Same extension, same command — Kiro is VS Code-compatible.

Download cfls.exe (no Node install needed). Then:

cfls.exe admin-init --team myteam
cfls.exe host --url wss://0.0.0.0:8730
cfls.exe agent --insecure-tls

First run of an unsigned build may show a Windows SmartScreen prompt — choose “Run anyway”.

Build everything from the repo (Node 20+ & pnpm):

git clone https://github.com/greninja-op/multi-agentic-workflow
pnpm install && pnpm -r build
pnpm -C apps/cli package:win

Produces apps/cli/dist-exe/cfls.exe. Package the extension with pnpm -C apps/vscode-extension package:vsix.

Team setup in 5 minutes

One admin sets up the host and invites devices. Each teammate joins with their own key.

Admin once per team

powershell
PS> cfls admin-init --team myteam
✓ admin key created & stored securely
PS> cfls host --url wss://0.0.0.0:8730
✓ CoordinationHost listening
PS> cfls invite bob <bob-public-key>
→ send the invitation string to Bob

Teammate on each laptop

powershell
PS> cfls clone <repo-url> --host wss://HOST-IP:8730 --name bob
PS> cfls id            # share this key with admin
PS> cfls connect <invitation>
PS> cfls agent --insecure-tls
✓ Online — open the repo in VS Code / Kiro
